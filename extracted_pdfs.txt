MASTER BLUEPRINT: FLOW REMBUSE WHUSCORE AI
PT CONNEXA (Whusnet) - Confidential Technical Document
1. TAHAP SUBMISSION (TEKNISI)
- Teknisi upload foto nota via WebApp.
- Input awal: Pilih Cabang & Metode Pembayaran (Cash, Transfer Teknisi, atau Transfer Vendor).
- Layer 1 (Security): Sistem melakukan "Image Hashing" untuk deteksi duplikat instan (<0.1 detik).
- Layer 2 (Logic): AI mengekstrak tanggal nota. Jika selisih > 2 hari dari hari ini, status otomatis
"AUTO-REJECT".
- Layer 3 (AI Extraction): AI mengisi otomatis field: Material, Jumlah, Satuan, Nominal, dan Nama Vendor.
- Status Akhir: "Pending - Menunggu Approval Admin".
2. TAHAP VERIFIKASI & APPROVAL (ADMIN/OWNER)
- Dashboard menampilkan isian AI dengan tanda khusus (High Confidence/Low Confidence).
- Logic Override: Jika nota berstatus "Auto-Reject" (telat > 2 hari), tombol "Request Override" hanya muncul
di level Admin/Owner.
- Requirement: Wajib mengisi kolom "Alasan Pengecualian" untuk setiap aksi Override.
- Keputusan: Jika disetujui, status berubah menjadi "Menunggu Pembayaran".
3. TAHAP PEMBAYARAN (3 METODE)
A. CASH (BAYAR LANGSUNG):
   - Admin upload foto penyerahan (Wajah Teknisi + Uang/Voucher).
   - Status: "Menunggu Konfirmasi Teknisi".
   - Closing: Teknisi klik "Terima" di Web atau via Telegram Bot untuk status "Selesai".
B. TRANSFER (TEKNISI / VENDOR):
   - Admin pilih rekening (Profil Tech / Input Manual Vendor).
   - Admin upload Bukti Transfer (Screenshot M-Banking).
   - AI OCR Verification: AI membaca nominal pada struk + Biaya Admin + Kode Unik.
   - Jika MATCH: Status otomatis "Selesai".
   - Jika MISMATCH: Status "Flagged - Selisih Nominal".
4. HANDLING SELISIH & FORCE APPROVEMASTER BLUEPRINT: FLOW REMBUSE WHUSCORE AI
PT CONNEXA (Whusnet) - Confidential Technical Document
- Status "Flagged" mengunci transaksi (tombol selesai mati).
- Notifikasi "Real-time" dikirim ke Telegram Owner jika ada selisih.
- Hanya Owner/Admin yang bisa melakukan "Force Approve" dengan alasan tertulis.
- Setiap selisih dicatat dalam database audit untuk laporan kebocoran bulanan.
5. TECH STACK RECOMMENDATION
- Backend: Laravel 11 + Redis (Queue Processing).
- AI Engine: RTX 3090 (Self-Hosted Ollama).
- Models: Llama-3.2-Vision (OCR) & DeepSeek-R1 (Reasoning/Audit).
- Automation: n8n (Telegram Bot Integration & Mutation Check).
- Database: PostgreSQL / MySQL with Audit Logging.

2. Authentication 
Endpoint publik (frontend → Laravel) tidak memerlukan autentikasi khusus. Endpoint callback (n8n → Laravel) 
dilindungi dengan header X-SECRET. 
 
2.1 Endpoint Publik 
Semua endpoint di /api/v1/ bersifat publik dan tidak memerlukan token. Diakses langsung oleh frontend aplikasi.  
 
POST /api/v1/nota/upload 
POST /api/v1/payment/cash/upload 
POST /api/v1/payment/cash/konfirmasi 
POST /api/v1/payment/transfer/upload 
GET  /api/v1/transaksi 
GET  /api/v1/transaksi/{id} 
 
2.2 Endpoint n8n Callback (X-SECRET Required) 
Endpoint callback hanya boleh dipanggil oleh n8n. Dilindungi middleware N8nSecretMiddleware yang memeriksa 
header X-SECRET. 
 
Header Value Keterangan 
X-SECRET mySuperSecretKey123 Nilai harus sama dengan config n8n dan 
.env Laravel 
Content-Type application/json Wajib untuk semua request callback 
 
Contoh Header Callback: 
POST /api/ai/auto-fill HTTP/1.1 
Host: localhost:8000 
Content-Type: application/json 
X-SECRET: mySuperSecretKey123 
 
PERHATIAN: Jika X-SECRET tidak ada atau salah, server akan mengembalikan 401 Unauthorized dan 
request ditolak.

3. Flow 1 — Upload Nota (3 Layer OCR) 
Flow utama sistem. Admin upload foto nota → Laravel forward ke n8n → 3 layer verifikasi berjalan di background 
→ callback hasil ke Laravel.

transfer nanti.

4. Flow 2 — CASH Payment 
Admin upload foto penyerahan (wajah teknisi + uang/voucher). n8n mengirim notifikasi Telegram ke teknisi. 
Teknisi mengkonfirmasi terima/tolak via Web atau Telegram Bot.

5. Flow 3 — TRANSFER Payment (AI OCR Verify) 
Admin upload screenshot M-Banking. n8n menggunakan Gemini AI untuk membaca nominal + biaya admin + 
kode unik dari struk. Hasilnya dibandingkan dengan expected_total. Toleransi selisih: Rp 500.  
 
Formula: expected_total = expected_nominal + biaya_admin + kode_unik 
 
 POST   /api/v1/payment/transfer/upload   Upload Bukti Transfer 
Admin mengupload screenshot M-Banking. AI OCR Gemini akan membaca nominal transfer, biaya admin, dan 
kode unik dari gambar lalu membandingkan dengan nilai yang diharapkan. Jika selisih <= Rp 500 maka MATCH 
dan status otomatis Selesai. 
 
Request — multipart/form-data 
Parameter Tipe Wajib Keterangan 
bukti_transfer file (image) Ya Screenshot M-Banking. JPG/PNG/HEIC. Maks 10MB. 
upload_id string Ya upload_id dari transaksi yang akan dibayar. 
transaksi_id string Ya transaksi_id dari transaksi. 
expected_nominal numeric Ya Nominal nota dalam rupiah. Contoh: 1320000 
kode_unik numeric Tidak Kode unik transfer (3 digit). Default: 0 
biaya_admin numeric Tidak Biaya admin bank. Contoh: 6500. Default: 0 
rekening_tujuan string Tidak Nomor rekening tujuan transfer. 
nama_bank_tujuan string Tidak Nama bank tujuan. Contoh: BCA, Mandiri. 
 
Response — 202 Accepted 
{ 
  "success": true, 
  "message": "Bukti transfer diterima. AI sedang memverifikasi nominal.",

6. n8n Callback Endpoints 
Endpoint berikut dipanggil oleh n8n, bukan oleh frontend. Selalu sertakan header X-SECRET. 
 POST   /api/ai/auto-fill   Callback Hasil OCR Layer 3 
Dikirim oleh n8n setelah Layer 3 (Gemini OCR) selesai atau jika terjadi reject di layer 1/2. Berisi status, 
confidence score per field, dan data hasil ekstraksi. 
 
Request Body — application/json 
Parameter Tipe Wajib Keterangan 
upload_id string Ya Identifier upload yang sedang diproses. 
status string Ya Nilai: success / auto_reject / low_confidence / failed 
confidence_label string Tidak HIGH (>70) atau LOW (<=70). Ada jika status = success. 
overall_confidence numeric Tidak Score confidence keseluruhan 0-100. 
field_confidence object Tidak Object berisi confidence per field: vendor, material, 
jumlah, satuan, nominal. 
vendor string Tidak Nama vendor hasil OCR. 
tanggal string Tidak Tanggal nota format DD/MM/YYYY. 
total_belanja numeric Tidak Total nominal nota. 
items array Tidak Array item barang dengan confidence per field. 
reason string Tidak Alasan reject/error. Ada jika status = auto_reject / failed. 
stage string Tidak Stage saat reject: layer1_security / layer2_logic / 
layer3_parse.

7. HTTP Status Codes & Error Handling 
Berikut adalah daftar lengkap HTTP status code yang digunakan oleh API ini beserta artinya.  
 
Kode Status Keterangan 
202 Accepted Request diterima, proses berjalan di background (async). Ada 
polling_url untuk cek status. 
200 OK Request berhasil diproses sepenuhnya. 
401 Unauthorized Header X-SECRET tidak ada atau salah. Hanya untuk endpoint 
callback n8n. 
404 Not Found Transaksi atau resource dengan ID yang diberikan tidak 
ditemukan. 
422 Unprocessable Entity Validasi input gagal. Lihat field "errors" dalam response untuk 
detail. 
500 Internal Server Error Error tidak terduga di server Laravel. 
502 Bad Gateway Laravel tidak bisa menghubungi n8n. Cek apakah n8n berjalan 
dan URL benar.
