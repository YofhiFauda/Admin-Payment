## Tips untuk Teknisi

### 📸 Tips Foto Nota

**DO (Lakukan):**

1. **Foto Segera Setelah Belanja**
   ```
   ✅ Foto nota langsung di toko
   ✅ Jangan tunggu sampai rumah/kantor
   ✅ Nota masih fresh, tidak kusut
   ```

2. **Gunakan Cahaya yang Baik**
   ```
   ✅ Foto di tempat terang
   ✅ Hindari bayangan
   ✅ Jangan pakai flash langsung (bisa silau)
   ✅ Foto di luar ruangan (cahaya alami) lebih baik
   ```

3. **Fokus Tajam**
   ```
   ✅ Tap layar untuk fokus (HP)
   ✅ Pastikan teks terbaca jelas
   ✅ Jangan goyang saat foto
   ✅ Gunakan mode HDR jika ada
   ```

4. **Frame Penuh**
   ```
   ✅ Seluruh nota masuk frame
   ✅ Tidak terpotong
   ✅ Tidak terlalu banyak background
   ✅ Foto dari atas (90 derajat)
   ```

**DON'T (Jangan):**

1. ❌ Foto blur atau gelap
2. ❌ Nota terlipat atau kusut
3. ❌ Foto miring atau terpotong
4. ❌ Screenshot dari foto lain
5. ❌ Edit foto (crop, filter, dll.)

---

### ⏰ Tips Timing

**Upload Nota Segera:**
```
✅ Upload dalam 24 jam setelah belanja
✅ Jangan tunggu sampai akhir bulan
✅ Sistem reject nota > 2 hari (kecuali ada alasan valid)
```

**Manfaat Upload Cepat:**
- ✅ Tidak lupa detail transaksi
- ✅ Nota tidak hilang
- ✅ Approval lebih cepat
- ✅ Uang cair lebih cepat

---

### 📝 Tips Mengisi Form

**1. Cek Data OCR**
```
✅ Review data yang terisi otomatis
✅ Pastikan vendor benar
✅ Pastikan nominal benar
✅ Pastikan tanggal benar
✅ Edit jika ada yang salah
```

**2. Pilih Kategori yang Tepat**
```
✅ Instalasi: Untuk pekerjaan instalasi
✅ Pembelian Alat: Untuk beli tools/equipment
✅ Maintenance: Untuk perbaikan/perawatan
✅ Operasional: Untuk biaya operasional (bensin, tol, dll.)
```

**3. Pilih Cabang yang Benar**
```
✅ Pilih cabang yang menanggung biaya
✅ Jika untuk beberapa cabang, pilih yang paling dominan
✅ Atau konsultasi dengan Admin
```

**4. Tulis Keterangan yang Jelas**
```
✅ Jelaskan untuk apa pembelian ini
✅ Contoh: "Kabel untuk instalasi di rumah Pak Budi, Jl. Sudirman No. 123"
❌ Jangan: "Beli kabel" (tidak informatif)
```

---

### 💬 Tips Komunikasi

**Jika Transaksi Ditolak:**
```
1. Baca alasan penolakan dengan teliti
2. Perbaiki sesuai feedback
3. Jika tidak setuju, diskusi dengan Admin
4. Jangan langsung submit ulang tanpa perbaikan
```

**Jika Urgent:**
```
1. Hubungi Admin via Telegram/WhatsApp
2. Jelaskan kenapa urgent
3. Minta prioritas approval
4. Follow up jika belum ada respon dalam 2 jam
```

## Tips untuk Admin

### ✅ Tips Approval

**Review dengan Teliti:**
```
1. Cek foto nota (jelas & asli?)
2. Cek kewajaran harga (bandingkan dengan Price Index)
3. Cek kesesuaian dengan kebutuhan
4. Cek duplikasi (apakah nota ini sudah pernah disubmit?)
```

**Approve/Reject Cepat:**
```
✅ Target: Approve/reject dalam 1-2 hari kerja
✅ Jangan biarkan pending > 3 hari
✅ Jika ragu, diskusi dengan Atasan/Owner
✅ Jika urgent, prioritaskan
```

**Berikan Alasan yang Jelas (Jika Reject):**
```
✅ Tulis alasan yang konstruktif
✅ Jelaskan apa yang salah
✅ Berikan saran perbaikan
✅ Contoh: "Nota tidak jelas, mohon upload ulang dengan foto yang lebih terang"
❌ Jangan: "Ditolak" (tidak informatif)
```

---

### 💸 Tips Pembayaran

**Upload Bukti Bayar Segera:**
```
✅ Upload dalam 1-2 hari setelah approval
✅ Jangan tunggu sampai akhir bulan
✅ Teknisi menunggu uang, jangan buat mereka menunggu lama
```

**Pilih Metode yang Tepat:**
```
Transfer Bank:
✅ Untuk nominal besar (> Rp 500.000)
✅ Untuk keamanan & audit trail
✅ Cek biaya admin sebelum transfer

Cash:
✅ Untuk nominal kecil (< Rp 500.000)
✅ Untuk pembayaran cepat
✅ Pastikan Teknisi konfirmasi terima
```

**Tips Transfer:**
```
✅ Cek biaya admin bank
✅ Jika ada biaya admin, transfer lebih (nominal + biaya admin)
✅ Atau gunakan transfer gratis (sesama bank, m-banking)
✅ Foto struk dengan jelas
✅ Pastikan nominal di struk sesuai
```

---

### 📊 Tips Monitoring

**Cek Dashboard Setiap Hari:**
```
✅ Lihat transaksi pending
✅ Lihat transaksi yang di-flag
✅ Lihat statistik transaksi
✅ Lihat rincian biaya per cabang
```

**Prioritas:**
```
1. Transaksi yang di-flag (perlu Force Approve)
2. Transaksi pending > 2 hari
3. Transaksi urgent (koordinasi via Telegram)
4. Transaksi normal
```

## Tips untuk Atasan

### 📝 Tips Edit Pengajuan

**Full Edit dengan Bijak:**
```
✅ Edit hanya jika memang perlu
✅ Jelaskan alasan perubahan di catatan
✅ Koordinasi dengan Teknisi jika perubahan signifikan
✅ Sistem tracking perubahan (Dual-Version)
```

**Contoh Edit yang Baik:**
```
Original (Teknisi):
- Item: Router Brand A
- Harga: Rp 5.000.000

Edited (Atasan):
- Item: Router Brand B
- Harga: Rp 4.500.000
- Catatan: "Brand B lebih reliable dan lebih murah Rp 500.000"
```

---

### 🎯 Tips Monitoring

**Analisis Pengeluaran:**
```
✅ Review rincian biaya per cabang setiap minggu
✅ Identifikasi cabang dengan pengeluaran tinggi
✅ Analisis penyebab (banyak instalasi? banyak maintenance?)
✅ Diskusi dengan Owner untuk optimasi
```

**Monitoring Hutang:**
```
✅ Cek hutang antar cabang setiap minggu
✅ Koordinasi pelunasan hutang
✅ Jangan biarkan hutang menumpuk
✅ Target: Lunas dalam 1 bulan
```

## Tips untuk Owner

### 🔍 Tips Review

**Review Transaksi ≥ 1 Jt:**
```
✅ Review dalam 1-2 hari kerja
✅ Jangan biarkan pending terlalu lama
✅ Koordinasi dengan Admin/Atasan jika ada pertanyaan
✅ Approve dari mana saja (mobile-friendly)
```

**Review Anomali Harga:**
```
✅ Cek dashboard Price Index setiap minggu
✅ Review transaksi dengan harga anomali
✅ Investigasi penyebab (harga naik? vendor baru?)
✅ Lock harga referensi jika perlu
```

**Review Transaksi yang Di-flag:**
```
✅ Review segera (jangan > 1 hari)
✅ Investigasi penyebab selisih
✅ Koordinasi dengan Admin & Teknisi
✅ Force Approve jika memang wajar
```

---

### 📊 Tips Analisis

**Analisis Bulanan:**
```
✅ Export data di awal bulan berikutnya
✅ Analisis:
   - Total pengeluaran per cabang
   - Breakdown per kategori
   - Trend pengeluaran (naik/turun?)
   - Efisiensi (bandingkan dengan target)
✅ Diskusi dengan Management
✅ Buat action plan untuk optimasi
```

**Analisis Tahunan:**
```
✅ Review pengeluaran selama 1 tahun
✅ Identifikasi pola (bulan mana paling tinggi?)
✅ Bandingkan dengan tahun lalu
✅ Buat budget untuk tahun depan
```

---

### 🔐 Tips Keamanan

**Audit Trail:**
```
✅ Review Activity Log setiap minggu
✅ Cek aktivitas mencurigakan
✅ Cek transaksi yang di-Force Approve
✅ Cek transaksi yang di-Override
```

**Access Control:**
```
✅ Review user access setiap bulan
✅ Nonaktifkan akun yang tidak aktif
✅ Reset password user yang lupa
✅ Pastikan setiap user hanya punya 1 akun
```

## Tips Umum

### 🔐 Tips Keamanan

**Password:**
```
✅ Ganti password default segera
✅ Gunakan password yang kuat (min 8 karakter, kombinasi huruf, angka, simbol)
✅ Jangan share password ke orang lain
✅ Ganti password setiap 3-6 bulan
```

**Logout:**
```
✅ Logout setelah selesai (terutama di komputer umum)
✅ Jangan biarkan browser terbuka tanpa pengawasan
✅ Sistem auto-logout setelah 2 jam tidak aktif
```

## Common Mistakes

### ❌ Kesalahan Umum & Cara Menghindarinya

**1. Upload Nota Terlambat**
```
❌ Masalah: Upload nota > 2 hari, auto-reject
✅ Solusi: Upload dalam 24 jam setelah belanja
```

**2. Foto Nota Tidak Jelas**
```
❌ Masalah: OCR gagal, harus input manual
✅ Solusi: Foto di tempat terang, fokus tajam
```

**3. Lupa Ganti Password Default**
```
❌ Masalah: Keamanan akun lemah
✅ Solusi: Ganti password segera setelah login pertama
```

**4. Approve Transaksi Sendiri**
```
❌ Masalah: Conflict of interest, audit trail buruk
✅ Solusi: Minta Admin/Atasan/Owner lain yang approve
```

**5. Biarkan Pending Terlalu Lama**
```
❌ Masalah: Teknisi menunggu lama, cash flow terganggu
✅ Solusi: Approve/reject dalam 1-2 hari kerja
```

**6. Reject Tanpa Alasan Jelas**
```
❌ Masalah: Submitter bingung, submit ulang dengan kesalahan yang sama
✅ Solusi: Tulis alasan yang konstruktif & saran perbaikan
```

**7. Transfer Tanpa Cek Biaya Admin**
```
❌ Masalah: Nominal yang diterima kurang, transaksi di-flag
✅ Solusi: Cek biaya admin, transfer lebih atau gunakan transfer gratis
```

**8. Tidak Setup Telegram Bot**
```
❌ Masalah: Miss notifikasi penting (konfirmasi cash, approval urgent)
✅ Solusi: Setup Telegram Bot segera
```

## Pro Tips

### 🚀 Tips Lanjutan

**1. Gunakan Keyboard Shortcuts**
```
Ctrl + F: Search
Ctrl + R: Refresh
Esc: Close modal
Enter: Submit form
```

**2. Bookmark Halaman Penting**
```
✅ Dashboard
✅ Transaksi Pending
✅ Buat Transaksi
✅ Notifikasi
```

**3. Gunakan Filter untuk Efisiensi**
```
✅ Filter status: Pending (untuk approval)
✅ Filter status: Flagged (untuk review)
✅ Filter cabang: Untuk analisis per cabang
✅ Filter tanggal: Untuk laporan periode tertentu
```

**4. Export Data Secara Berkala**
```
✅ Export di awal bulan (untuk laporan bulan lalu)
✅ Simpan di folder terorganisir
✅ Gunakan untuk analisis & audit
```

**5. Manfaatkan Dashboard**
```
✅ Cek dashboard setiap hari (5 menit)
✅ Prioritaskan transaksi pending & flagged
✅ Monitor trend pengeluaran
✅ Identifikasi anomali segera
```

**6. Koordinasi Tim**
```
✅ Buat grup Telegram untuk koordinasi
✅ Share update penting di grup
✅ Diskusi masalah & solusi
✅ Saling remind jika ada pending lama
```

**7. Continuous Improvement**
```
✅ Review proses setiap bulan
✅ Identifikasi bottleneck
✅ Diskusi dengan tim untuk optimasi
✅ Implement improvement
```
